# Linux コマンドとBash変数のルール

このドキュメントは、Linuxで頻繁に使用されるコマンドと、Bashスクリプトにおける変数の基本的なルールをまとめたものです。

---

## 1. Bashにおける変数と環境変数

Bashスクリプトを記述する際に使用する「シェル変数」と「環境変数」のルールと使い方を整理します。

### ◆ 変数を定義する際のルール

一貫性のあるコードを書くための一般的なルールです。

-   **命名**:
    -   **英数字**と**アンダースコア `_`** を使用します。
    -   1文字目は**英字**または**アンダースコア**である必要があります（数字は使えません）。
    -   **大文字と小文字は区別されます**。
-   **書式**:
    -   **小文字**と**スネークケース**（例: `my_variable`）で変数を宣言するのが一般的です。
    -   値にスペースが含まれる場合は、**ダブルクォーテーション `""`** で囲みます。
    -   変数を呼び出す際は、`${変数名}` のように波括弧 `{}` で囲むことが推奨されます。（意図しない文字連結を防ぐため）
    -   `HOME` や `PATH` などの**既存の環境変数は上書きしない**よう注意が必要です。

### ◆ シェル変数 (Shell Variable)

現在のシェルセッション内でのみ有効な変数です。

-   **宣言方法**: `変数名="値"`
-   **値の種類**:
    -   **文字列**: `my_string="hello"`
    -   **コマンドの実行結果**: `current_date="$(date '+%Y-%m-%d')"`
    -   **計算結果**: `sum="$((10 + 5))"`
-   **呼び出し方法**: `echo "${変数名}"`

```bash
# シェル変数の使用例
type_string="hoge"
type_command="$(date '+%H:%M')"
type_calc="$((10 + 10))"

echo "文字列: ${type_string}"
echo "現在時刻: ${type_command}"
echo "計算結果: ${type_calc}"
