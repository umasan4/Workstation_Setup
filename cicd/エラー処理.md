#### ※ ステップ中のエラー処理
```yaml
#----------------------------------------
# 1).ステップ内の処理は止めずにすべて実行させる
#----------------------------------------
# shell: の引数に +e を指定する（エラー時でも中断しない）

jobs:
  #--- 省略 ---
  steps:
    - name: sample
      shell bash +e {0}
      run echo "Hello world"

# 使用ケース:
# 処理結果の exit codeを取り出すとき

# 注意:
# 実行結果は失敗になる
# これ以降のステップ、ジョブは止まる
```
```yaml
#----------------------------------------
# 2).エラーが発生しても無視させる
#----------------------------------------
# shell: の引数に continue-on-error: trueを指定する
  jobs:
      #--- 省略 ---
      steps:
        - name: sample
          continue-on-error: true
          run echo "Hello world"

# 使用ケース:
# 単体テストなど、あらかじめ失敗想定される場合に使用する

# 注意:
# 実行結果は成功になる
# これ以降のステップ、ジョブは継続される
```
```yaml
#----------------------------------------
# 3).指定ステップを必ず実行させたい
#----------------------------------------
# shell: の引数に if: always() を指定する
  jobs:
      #--- 省略 ---
      steps:
        - name: sample
          if: always()
          run echo "Hello world"

# 使用ケース:
# 通知処理、結果出力など必ず実行させたい処理に使用する
```
#### ※ ジョブ中のエラー処理
```yaml
#----------------------------------------
# 2).エラーが発生しても無視させる
#----------------------------------------
# 記述する場所がjobの中になるだけ
jobs:
  job1:
    name: Sample Job
    runs-on: ubuntu-latest
    continue-on-error: true
    steps:
    #--- 省略 ---

# 注意:
# ジョブの実行結果は成功になる
# なかで動作するステップがエラーとなると以降のステップは実行されない
# 後続のジョブは動作する
```
